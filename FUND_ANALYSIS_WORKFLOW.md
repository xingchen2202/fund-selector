# 基金分析标准流程 (Fund Analysis Workflow)

> 当用户询问"买什么基金"、"怎么买基金"、"推荐基金"、"XX 基金怎么样"时，必须按以下流程执行。

---

## Step 1: 宏观与大势研判 (Macro & Market Context)

**目标**：确定当前市场整体基调，决定进攻/防守策略。

### 调用工具

```
cn-financial MCP:
  - get_macro_pmi        → PMI 趋势（经济方向）
  - get_macro_money_supply → M0/M1/M2 增速（流动性）
  - get_market_overview   → 主要指数行情（当前水位）
  - get_north_bound_flow  → 北向资金流向（外资情绪）
  - get_sector_fund_flow  → 板块资金流向（主力方向）
```

### 思考问题

1. PMI > 50 且上升 → 经济复苏；PMI < 50 且下降 → 经济衰退
2. M2 增速 > 10% → 流动性宽松；M2 < 8% → 流动性收紧
3. 北向资金连续净流入 → 外资看多；连续净流出 → 外资谨慎
4. 指数估值分位 < 30% → 低估区域，适合进攻；> 70% → 高估区域，适合防守

### 决策输出

确定整体基调，例如：
- "当前经济复苏+流动性宽松，适合进攻，股基占比可提高"
- "当前经济过热+通胀抬头，适合防守，高股息+债券为主"
- "当前市场震荡+信号不明，建议均衡配置+定投"

---

## Step 2: 资产配置与赛道选择 (Asset Allocation & Sector)

**目标**：根据宏观判断，确定股/债/现金比例，选择景气方向。

### 调用工具

```
cn-financial MCP:
  - get_industry_list     → 申万行业板块涨跌幅
  - get_sector_fund_flow  → 行业/概念资金流向
  - get_bond_yield_curve  → 国债收益率曲线（无风险利率锚）
```

### 思考问题

1. 哪些行业板块近期资金持续流入？（主力关注方向）
2. 哪些行业处于景气上行周期？（结合 PMI 和产业政策）
3. 当前利率环境如何？（利率下行 → 债券牛市；利率上行 → 债券谨慎）

### 决策输出

确定配置比例，例如：
- 股票型：60%（其中宽基 40% + 行业 20%）
- 债券型：30%
- 货币/现金：10%

确定景气赛道，例如：
- 当前看好：AI/半导体（科技成长）、高股息（防御）
- 当前回避：房地产、传统能源

---

## Step 3: 基金筛选与穿透分析 (Fund Screening & Deep Dive)

**目标**：从全市场筛选出符合条件的基金，并穿透验证底层资产质量。

### 3.1 初筛基金池

```
cn-mutual-fund MCP:
  - get_fund_ranking(fund_type)     → 获取业绩排行候选池
  - get_fund_info(fund_code)        → 获取规模、经理、费率、类型
  - get_fund_nav_history(fund_code) → 获取净值，计算最大回撤/夏普
```

### 筛选条件（根据用户风险偏好调整）

| 维度 | 保守型 | 稳健型 | 进取型 |
|------|--------|--------|--------|
| 最大回撤 | < 10% | < 20% | < 30% |
| 规模 | 10-200 亿 | 5-500 亿 | 2-1000 亿 |
| 经理任职 | > 2 年 | > 1 年 | > 1 年 |
| 晨星评级 | ≥ 4 星 | ≥ 3 星 | 无要求 |

### 3.2 穿透分析（验证底层资产）

对筛选出的 Top 5-10 候选基金：

```
cn-mutual-fund MCP:
  - get_fund_portfolio(fund_code)   → 十大重仓股/债券配置

cn-financial MCP（对重仓股逐一验证）:
  - get_company_profile(stock_code)       → 公司主营业务
  - get_financial_indicators(stock_code)  → ROE/毛利率/资产负债率
  - get_valuation_metrics(stock_code)     → PE/PB 估值水平
```

### 思考问题

1. 重仓股是否集中在景气行业？还是逆配置？
2. 重仓股财务质量如何？（ROE > 15%？负债率合理？）
3. 重仓股估值是否合理？（PE 历史分位？）
4. 基金持仓与宣称风格是否一致？（有没有挂羊头卖狗肉）

### 决策输出

筛选出 3-5 只最终候选基金，附带穿透分析结论。

## Step 4: 制定买卖与资金计划 (Execution Strategy)

**目标**：摒弃"全仓买入"，根据波动率和估值生成具体执行方案。

### 4.1 计算关键价位

基于 `get_fund_nav_history` 近 3-6 个月数据：

```
关键指标：
- 近 3 个月波动率 = 日收益标准差 × √252
- 近期支撑位 = 近 3 个月净值低点
- 近期压力位 = 近 3 个月净值高点
- 当前净值在历史分位 = 当前 PE / 近 5 年 PE 均值
```

### 4.2 生成执行计划

根据用户资金量和风险偏好，选择一种策略：

#### 策略 A：DCA 定投（推荐默认）

```
总资金：{amount} 万元
单期金额 = 总资金 / 期数

触发条件：
- 估值 < 30% 分位 → 加倍定投（×1.5）
- 估值 30%-70% 分位 → 正常定投
- 估值 > 70% 分位 → 暂停定投，考虑止盈

止盈纪律：
- 浮盈 15% → 卖出 1/3
- 浮盈 25% → 卖出 1/3
- 浮盈 35% → 清仓

止损纪律：
- 浮亏 15% → 暂停定投，复盘策略
```

#### 策略 B：金字塔建仓

```
底仓：30% → 现价直接买入
加仓 1：30% → 净值跌 5% 时触发
加仓 2：20% → 净值跌 10% 时触发
加仓 3：20% → 净值跌 15% 时触发

止损：跌破 -20% 停止加仓，等待企稳
```

#### 策略 C：网格交易（仅适用于 ETF）

```
网格间距 = 年化波动率 / √12（约 1 个月波动）
每格金额 = 总资金 / 网格数量

买入价 = 当前净值 - N × 网格间距
卖出价 = 当前净值 + N × 网格间距
```

## Step 5: 压力测试与最坏情景（Pressure Test）

**目标**：评估推荐组合在极端行情下的表现，避免"历史业绩好就推荐"的陷阱。

### 5.1 极端行情回撤估计

基于历史极端行情（2015 股灾、2020 疫情、2022 下跌）：
```
估计指标：
- 组合最大可能回撤 = 各基金历史最大回撤的加权平均 × 1.2（极端系数）
- 最坏情景下的定投承受能力：若浮亏 >25%，定投计划是否还能持续
- 极端行情触发条件："若组合浮亏 >25%，启动应急预案（暂停加仓+复盘）"
```

### 5.2 持仓相关性警告

```
检查项：
- 若两只基金的前十大重仓股有 ≥3 只相同 → 警告"持仓高度重叠，分散效果有限"
- 若行业配置相关系数 >0.8 → 建议增加低相关性资产
```

### 5.3 输出格式

```
🔴 压力测试
- 极端行情估计回撤：-XX% ~ -XX%
- 最坏情景承受力：可承受 / 需降低仓位
- 持仓相关性警告：[如有]
- 应急预案：浮亏 >25% → 暂停加仓 + 季度复盘
```

## Step 6: 策略回测（可选）

**目标**：用历史数据验证推荐策略的有效性。

```
基于近 3-5 年历史净值，回测推荐策略：
- DCA 定投的年化收益 / 最大回撤 / 夏普比率
- 与一次性买入的对比
- 最坏入场时机的表现（如 2015 年高点入场）
```

---

## 执行约束

1. **必须按顺序执行**：宏观 → 配置 → 筛选 → 执行 → 压力测试，不可跳过
2. **穿透验证不可省略**：选主动基金必须看十大重仓股，验证底层质量
3. **不做全仓建议**：永远分批建仓，永远设止损
4. **被动 vs 主动要区分**：
   - 被动基金（指数/ETF）：重点看跟踪误差、费率、规模
   - 主动基金：重点看经理风格、稳定性、超额收益
5. **始终附带免责声明**
6. **数据不足时降级**：如果 akshare 接口返回不足，标注"数据缺失"而非编造
