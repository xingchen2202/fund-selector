## 问题详情

### P0-1：持仓穿透降级

**现状**：`get_fund_portfolio` 返回 `stock_holdings: []`，无法穿透到个股  
**修复**：增加三级穿透降级规则（A/B/C 级），持仓缺失时引导用户查阅基金季报  
**验证断言**：B2-1

### P0-2：费率多源获取

**现状**：所有基金的 `费率详情` 均为 `{}`  
**修复**：增加多源获取逻辑（MCP → 东方财富 → 标注缺失 + 引导销售平台）  
**验证断言**：B2-2

### P1-1：风险指标交叉验证

**现状**：MCP 返回"近 1 年最大回撤=7%"，实际净值计算为 -32.96%  
**修复**：增加交叉验证规则，偏差 > 5pp 时标注 [⚠️ 数据异常] 并优先采用独立计算值  
**验证断言**：B2-3

### P1-2：经理查询拆分

**现状**：`get_fund_manager_info("张坤 彭珂")` 失败  
**修复**：增加组合名自动拆分 + 分别查询 + 合并输出规则  
**验证断言**：B2-4

### P2：估值分位查询

**现状**：无法获取指数 PE/PB 分位数据  
**修复**：接入 cn-financial 的 get_valuation_metrics，增加市场温度判定和仓位建议  
**验证断言**：B2-5

### P1-3：经理主理人判定防呆

**现状**：MCP 返回的经理顺序不稳定（2026-07-23 "郑晓辉 刘睿聪"，2026-07-24 "刘睿聪 郑晓辉"）。若模型按 MCP 顺序取第一个为主理人，会误判刘睿聪（1304 天）为主理人，实际应为郑晓辉（5337 天）  
**修复**：强化经理查询容错规则，显式禁止按 MCP 返回顺序判定，必须查询比较从业天数，输出附注判定依据  
**验证断言**：B3-1

### P2-1：估值工具失败降级

**现状**：`get_valuation_metrics("000300")` 返回空数据，规则缺少工具失败时的降级路径  
**修复**：增加 3 级降级链（get_valuation_metrics → 其他 PE/PB 工具 → PMI/M2/北向资金），严禁编造估值分位数字  
**验证断言**：B3-2, B3-3

## 真实使用案例

### 案例 1：正常基金分析（#1）
- **输入**：帮我分析基金 000001
- **触发问题**：持仓明细为空、费率缺失
- **修复后表现**：输出穿透深度评级（B 级），引导用户查季报

### 案例 2：风险指标矛盾（#9）
- **输入**：110011 近 1 年最大回撤
- **触发问题**：MCP 返回 7%，实际 -32.96%
- **修复后表现**：标注 [⚠️ 数据异常]，展示两个值并说明差异

### 案例 3：经理查询失败（#8）
- **输入**：000001 的基金经理
- **触发问题**：组合名"郑晓辉 刘睿聪"查询失败
- **修复后表现**：自动拆分查询，正确输出两位经理信息

### 案例 4：经理主理人判定防呆（iteration-3 重测）
- **输入**：帮我分析基金 000001 的基金经理，谁是主理人
- **触发问题**：MCP 返回顺序"刘睿聪 郑晓辉"（刘在前），若按顺序取第一个会误判
- **修复后表现**：显式引用规则"禁止以 MCP 返回顺序判定主理人"，查询比较从业天数（郑晓辉 5337 天 > 刘睿聪 1304 天），正确判定郑晓辉为主理人，附注判定依据

### 案例 5：估值工具失败降级（iteration-3 重测）
- **输入**：帮我选基金，当前沪深300 估值分位多少
- **触发问题**：get_valuation_metrics("000300") 返回空数据
- **修复后表现**：完整执行 3 级降级链（get_valuation_metrics → 3 个备选工具 → PMI 50.3 + M2 8.0%），正确剔除 2014 年北向资金陈旧数据，输出 [⚠️ 估值数据不可用]，未编造任何分位数字

## 已知限制

| 限制 | 原因 | 影响 |
|------|------|------|
| AKShare 基金估值不可用 | 数据源不支持基金级别估值 | B2-5 无法获取真实估值数据 |
| 持仓明细恒为空 | AKShare MCP 限制 | 穿透深度最高为 B 级 |
| 费率详情恒为空 | AKShare MCP 限制 | 费率对比功能受限 |
