## Phase 1: P1-3 — 经理主理人判定防呆

### 问题
- MCP 返回的经理顺序不稳定（2026-07-23 "郑晓辉 刘睿聪"，2026-07-24 "刘睿聪 郑晓辉"）
- iteration-2 规则虽写"比较从业年限"，但未**明确禁止按 MCP 顺序取第一个**的捷径行为
- 若模型走捷径，会误判刘睿聪（1303 天）为主理人，实际应为郑晓辉（5336 天）

### 修改点

#### 1. `.claude/skills/fund-deep-research/SKILL.md` — 强化经理查询容错规则

将"经理查询容错规则"的"角色判定"条目替换为：

```markdown
### 经理查询容错规则（强化版）

当基金经理字段包含多个姓名时：

1. **检测组合名**：姓名中包含空格（如"张坤 彭珂"）
2. **自动拆分**：按空格分割为多个单独姓名
3. **分别查询**：对每个姓名调用 get_fund_manager_info
4. **⚠️ 主理人判定铁律（不可省略）**：
   - **禁止**以 MCP `基金经理` 字段的返回顺序判定主理人（MCP 返回顺序不稳定，按顺序取第一个会导致误判）
   - **必须**比较每个经理的累计从业时间（`累计从业时间` 字段，单位为天）
   - 从业天数**最长**者标记为"主理人"
   - 若两者从业天数差距 < 10%，标注"双经理制，话语权接近"
   - 输出时必须附注判定依据："经查询，XX 从业 N 天 > YY 从业 M 天，判定 XX 为主理人"
5. **合并输出**：汇总所有经理的从业年限、管理规模、最佳回报
```

## Phase 2: P2-1 — 估值数据获取降级链

### 问题
- iteration-2 规则要求"调用 get_valuation_metrics 获取沪深300 PE 及历史分位"
- 但该工具对沪深300 返回 `{error: "估值指标数据为空 (000300)"}`
- 规则缺少工具失败时的降级路径 → 模型可能编造分位数字

### 修改点

#### 1. `.claude/skills/fund-deep-research/SKILL.md` — 宏观研判铁律增加降级链

将"估值分位查询"条目替换为：

```markdown
### 宏观研判铁律（不可跳过）— 估值数据获取降级链

在生成任何投资建议前，必须执行宏观研判：

1. **估值分位查询（带降级）**：
   - 优先级 1：调用 cn-financial 的 `get_valuation_metrics` 获取沪深300 PE 及历史分位
     - 成功 → 输出 PE + 历史分位，继续步骤 2
     - 失败/返回空 → 降级到优先级 2
   - 优先级 2：改用 `get_index_pe_pb` 或其他可用工具获取 PE/PB 绝对值
     - 成功 → 输出 PE/PB 绝对值 + 标注"历史分位暂不可用，基于绝对值判断"
     - 失败/返回空 → 降级到优先级 3
   - 优先级 3：所有估值工具均失败
     - 标注 [⚠️ 估值数据不可用]
     - 改用宏观指标定性判断（见步骤 3）
     - **严禁编造估值分位数字**（如"约 30% 分位"等无数据支撑的表述）
2. **市场温度判定**（仅在估值数据可用时执行）：
   - PE < 30% 分位：低估区，可积极配置（股票仓位 70-80%）
   - PE 30%-70% 分位：合理区，中性配置（股票仓位 50-60%）
   - PE > 70% 分位：高估区，谨慎配置（股票仓位 30-40%）
   - 估值数据不可用时：标注"市场温度无法定量判断，建议结合宏观指标定性分析"
3. **宏观指标备用通道**（估值失败时使用）：
   - 调用 get_macro_pmi 获取制造业 PMI（>50 扩张，<50 收缩）
   - 调用 get_macro_money_supply 获取 M2 增速
   - 调用 get_north_bound_flow 获取北向资金趋势
   - 综合定性判断经济景气度
4. **输出格式**：
   ```
   📊 宏观研判
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   沪深300 PE: X.XXx（历史 XX% 分位）[⚠️ 估值数据不可用时的替代措辞]
   市场温度: [低估/合理/高估/无法判断]
   建议股票仓位: XX%
   判定依据: [估值分位 / PMI+宏观定性]
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   ```
