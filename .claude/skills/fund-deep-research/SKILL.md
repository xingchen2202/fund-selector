对单只基金进行全面深度研究，输出结构化分析报告。

## 触发
"XX 基金怎么样"、"深度研究 XX"、"全面分析 XX"

## 流程
1. MCP 获取基金基本信息、净值历史、经理信息、十大持仓
2. 穿透分析：重仓股财务质量 + 行业配置 + 风格稳定性
3. 六关评分（能力圈/经济特征/护城河/管理层/安全边际/估值）
4. 镜子测试（5 句话说清投资逻辑）
5. 反向测试（若判断错误，最可能原因）

## 输出
- 基本面：规模/经理/费率/业绩/回撤
- 穿透：重仓股质量 + 行业集中度
- 六关评分（★1-5）+ 综合评级
- 风险提示 + 逆向思考

## 工具依赖
- `mcp__cn-mutual-fund` — 基金信息/净值/经理/持仓获取
- `mcp__cn-financial` — A股行情/宏观/行业数据
- `tools/financial_rigor.py` — Decimal 精度验算（verify-scale/cross-validate）

## 数据质量与容错规则

### 持仓穿透降级规则

当 get_fund_portfolio 返回 stock_holdings 为空时：

1. **一级穿透**（行业配置）：使用 industry_allocation 字段，分析前三大行业占比
2. **二级穿透**（个股明细）：标注 [⚠️ 持仓明细缺失]，建议用户：
   - 查阅基金最新季报（天天基金/东方财富）
   - 关注前十大重仓股是否与基金契约一致
3. **穿透深度评级**：
   - A 级：获取到前十大重仓股 + 行业配置
   - B 级：仅获取到行业配置
   - C 级：仅获取到穿透数据为空，需人工验证

### 费率多源获取规则

当 get_fund_info 返回费率详情为空时：

1. **优先级 1**：MCP get_fund_info → 提取管理费/托管费/申购费/赎回费
2. **优先级 2**：尝试东方财富/天天基金 API 获取费率
3. **优先级 3**：标注 [⚠️ 费率缺失]，引导用户：
   - 基金公司官网 → 基金公告/招募说明书
   - 销售平台（天天基金/蚂蚁/银行）→ 费率公示
4. **费率对比**：同类型基金横向对比时，仅在费率数据完整时纳入对比

### 风险指标交叉验证规则

当 get_fund_info 返回风险指标时：

1. **独立计算**：用净值历史计算近 1 年最大回撤
   - 最大回撤 = (峰值 - 谷值) / 峰值
   - 峰值 = 区间内最高净值，谷值 = 峰值之后的最低净值
2. **对比验证**：独立计算值 vs MCP 返回值
   - 偏差 ≤ 5pp：采用 MCP 值（标注"已验证"）
   - 偏差 > 5pp：采用独立计算值（标注 [⚠️ 数据异常]）
3. **报告输出**：展示两个值，说明差异原因

### 经理查询容错规则（强化版）

当基金经理字段包含多个姓名时：

1. **检测组合名**：姓名中包含空格（如"张坤 彭珂"）
2. **自动拆分**：按空格分割为多个单独姓名
3. **分别查询**：对每个姓名调用 get_fund_manager_info
4. **⚠️ 主理人判定铁律（不可省略）**：
   - **禁止**以 MCP `基金经理` 字段的返回顺序判定主理人（MCP 返回顺序不稳定，按顺序取第一个会导致误判）
   - **必须**比较每个经理的累计从业时间（`累计从业时间` 字段，单位为天）
   - 从业天数**最长**者标记为"主理人"
   - 若两者从业天数差距 < 10%，标注"双经理制，话语权接近"
   - 输出时必须附注判定依据："经查询，XX 从业 N 天 > YY 从业 M 天，判定 XX 为主理人"
5. **合并输出**：汇总所有经理的从业年限、管理规模、最佳回报

### 宏观研判铁律（不可跳过）— 估值数据获取降级链

在生成任何投资建议前，必须执行宏观研判：

1. **估值分位查询（带降级）**：
   - 优先级 1：调用 cn-financial 的 `get_valuation_metrics` 获取沪深300 PE 及历史分位
     - 成功 → 输出 PE + 历史分位，继续步骤 2
     - 失败/返回空 → 降级到优先级 2
   - 优先级 2：改用 `get_index_pe_pb` 或其他可用工具获取 PE/PB 绝对值
     - 成功 → 输出 PE/PB 绝对值 + 标注"历史分位暂不可用，基于绝对值判断"
     - 失败/返回空 → 降级到优先级 3
   - 优先级 3：所有估值工具均失败
     - 标注 [⚠️ 估值数据不可用]
     - 改用宏观指标定性判断（见步骤 3）
     - **严禁编造估值分位数字**（如"约 30% 分位"等无数据支撑的表述）
2. **市场温度判定**（仅在估值数据可用时执行）：
   - PE < 30% 分位：低估区，可积极配置（股票仓位 70-80%）
   - PE 30%-70% 分位：合理区，中性配置（股票仓位 50-60%）
   - PE > 70% 分位：高估区，谨慎配置（股票仓位 30-40%）
   - 估值数据不可用时：标注"市场温度无法定量判断，建议结合宏观指标定性分析"
3. **宏观指标备用通道**（估值失败时使用）：
   - 调用 get_macro_pmi 获取制造业 PMI（>50 扩张，<50 收缩）
   - 调用 get_macro_money_supply 获取 M2 增速
   - 调用 get_north_bound_flow 获取北向资金趋势
   - 综合定性判断经济景气度
4. **输出格式**：
   ```
   📊 宏观研判
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   沪深300 PE: X.XXx（历史 XX% 分位）[⚠️ 估值数据不可用时的替代措辞]
   市场温度: [低估/合理/高估/无法判断]
   建议股票仓位: XX%
   判定依据: [估值分位 / PMI+宏观定性]
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   ```

## 失败处理
- MCP 超时/异常 → 标注"数据不可用"并继续，不中止整体流程
- MCP 返回陈旧数据（如 2014 年北向资金）→ 标注异常跳过该维度
- 全部 MCP 失败 → 输出"当前无法获取实时数据，请稍后重试"
