# A股财报分析工作流

## 工作流总览

```
财报发布 → 数据采集 → 业绩速评 → 深度分析 → 预测调整 → 估值更新 → 报告输出
```

## 步骤详解

### 步骤 1：数据采集（10分钟）

**通过 cn-financial-mcp 获取：**
```
1. get_income_statement(symbol, period="quarterly")  → 利润表
2. get_balance_sheet(symbol, period="quarterly")      → 资产负债表
3. get_cash_flow(symbol, period="quarterly")          → 现金流量表
4. get_financial_indicators(symbol)                    → 财务指标
5. get_realtime_quote(symbol)                          → 当前股价
6. get_company_info(symbol)                            → 公司基本信息
7. get_stock_news(symbol)                              → 近期新闻
```

**计算单季度数据：**
- Q2 = H1 - Q1
- Q3 = 三季报 - H1
- Q4 = 年报 - 三季报
- 注意：部分科目不能简单相减（如资产负债表为时点数）

### 步骤 2：业绩速评（5分钟）

快速判断框架：

| 维度 | 超预期 | 符合预期 | 低于预期 |
|------|--------|---------|---------|
| 营收增速 | > 一致预期 5% | ±5% 以内 | < 一致预期 -5% |
| 归母净利 | > 一致预期 10% | ±10% 以内 | < 一致预期 -10% |
| 毛利率 | > 去年同期 +1pp | ±1pp 以内 | < 去年同期 -1pp |
| 经营现金流 | > 净利润 | 0.5x - 1.0x 净利润 | < 0.5x 净利润 |

### 步骤 3：深度分析（20分钟）

#### 3.1 营收拆分
- 按产品/业务线拆分
- 按区域拆分（国内/海外，区域结构）
- 量价拆分（销量 × 单价）
- 新客户 vs 老客户

#### 3.2 成本分析
- 原材料成本变动
- 人工成本变动
- 折旧摊销变动
- 产能利用率

#### 3.3 费用分析
- 销售费用：广告投放、渠道建设
- 管理费用：人员扩张、股权激励摊销
- 研发费用：资本化比例、在研项目
- 财务费用：借款利率、汇兑损益

#### 3.4 关键科目异动
- 应收账款/合同资产增速 vs 营收增速
- 存货增速 vs 成本增速
- 预付/预收款项变化
- 减值损失计提

#### 3.5 现金流质量
- 经营现金流/净利润比
- CAPEX 强度（CAPEX/营收）
- 自由现金流趋势
- 分红能力（自由现金流 vs 分红支出）

### 步骤 4：盈利预测调整（10分钟）

**调整依据：**
1. 本期实际 vs 之前预测的差异
2. 管理层指引变化
3. 行业景气度变化
4. 一次性项目剔除

**调整项目：**
```
营收预测 → 毛利率假设 → 费用率假设 → 非经常性损益 → 税率
→ 净利润预测 → EPS → 目标价
```

**交叉验证：**
- 预测的营收增速是否与行业增速匹配
- 预测的利润率是否与产能利用率匹配
- 预测的 CAPEX 是否与扩产计划匹配
- 预测的经营现金流是否与利润匹配

### 步骤 5：估值更新（5分钟）

**获取可比数据：**
```
1. get_valuation_metrics(symbol)                       → 当前估值
2. get_industry_stocks(industry)                       → 同行业公司
3. get_historical_price(symbol, period="monthly")      → 历史估值
```

**估值方法选择：**
| 行业类型 | 首选方法 | 辅助方法 |
|---------|---------|---------|
| 消费/医药 | PE/PEG | DCF |
| 银行/保险 | PB-ROE | DDM |
| 科技成长 | PS/PE（远期） | DCF |
| 周期制造 | PB/EV/EBITDA | 重置成本 |
| 公用事业 | DDM/DCF | PE |
| 地产 | NAV | PB |

### 步骤 6：报告输出

参见 `report-structure.md` 获取详细报告模板。

**输出检查清单：**
- [ ] 核心结论明确（是否调整评级/目标价）
- [ ] 关键数据与公告一致
- [ ] 单季度数据计算正确
- [ ] 同比/环比方向一致
- [ ] 盈利预测调整有明确理由
- [ ] 估值方法与行业匹配
- [ ] 风险提示覆盖关键不确定性
- [ ] 数据来源标注完整

## 季报日历参考

| 报告期 | 披露窗口 | 重点关注 |
|--------|---------|---------|
| 一季报 | 4月1日-30日 | 开年经营趋势，是否受春节影响 |
| 半年报 | 7月1日-8月31日 | 上半年完整经营数据，可选审阅 |
| 三季报 | 10月1日-31日 | 前三季度趋势，Q4展望 |
| 年报 | 1月1日-4月30日 | 全年业绩，审计意见，分红方案 |
| 业绩预告 | 各季度报告前 | 提前判断业绩方向 |

## 同行业横向比较框架

在分析单家公司时，须同步获取 2-3 家核心可比公司的同期数据：

```
get_competitors(symbol)  → 同行业竞争对手
→ 对每家竞争对手:
   get_income_statement(comp_symbol)
   get_valuation_metrics(comp_symbol)
```

比较维度：
1. 营收增速排名
2. 毛利率/净利率排名
3. 费用率对比
4. ROE 对比
5. 估值溢价/折价合理性
