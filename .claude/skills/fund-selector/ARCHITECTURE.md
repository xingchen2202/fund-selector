# Fund Selector v2.0 — 架构说明文档

> 基于 ai-berkshire (MIT) 三层架构哲学重构的 A 股公募基金投研助手。

---

## 一、设计理念

### 核心哲学（移植自 ai-berkshire）

1. **对抗式多视角**：不做单一视角判断，让 4 个大师视角独立分析、互相挑战（当前为文档驱动模式：生成 4 视角 prompt 由 Claude 独立分析，未来可升级为真多 Agent 并行）
2. **数据严谨性**：每个关键数据点强制双源交叉验证，>1% 偏差自动报警
3. **买入后纪律**：不止于推荐，更包含追踪、漂移检测、卖出纪律
4. **报告可审计**：15% 随机抽样验证，确保每份报告的数据可追溯

### 三层架构

```
┌─────────────────────────────────────────────────────────┐
│  Skill 层（19 个入口）                                    │
│  深度研究 │ 财报分析 │ 行业筛选 │ 持仓管理 │ 思维工具       │
│  用户意图 → 路由表 → 选用对应 skill                         │
├─────────────────────────────────────────────────────────┤
│  Agent 层（团队型 skill 专用）                              │
│  Team Lead 并行调度 4 大师视角 → 冲突检测 → 综合研判          │
│  轻量 skill 不经过此层，直连工具快进快出                      │
├─────────────────────────────────────────────────────────┤
│  工具层（5 个工具）                                        │
│  精确计算 │ 实时检索 │ 报告抽检 │ 双源验证 │ 动量筛选        │
└─────────────────────────────────────────────────────────┘
```

## 三、Agent 层

### 4 大师视角

| Agent | 视角 | 核心问题 | 评分维度 |
|-------|------|---------|---------|
| 价值 Agent | 巴菲特 | 财务质量如何？估值便宜吗？ | 规模/费率/经理稳定性 |
| 成长 Agent | 段永平 | 商业模式好吗？护城河宽吗？ | 收益/动量/赛道景气 |
| 风控 Agent | 李录 | 最大风险是什么？管理层可信吗？ | 回撤/波动/极端亏损 |
| 周期 Agent | 芒格 | 行业格局如何？竞争态势怎样？ | 行业周期+逆向思考 |

### 调度流程

```
用户触发团队型 skill
  → Team Lead 并行启动 4 个 Agent
  → 各自独立 MCP 搜索 + 独立判断
  → 4 视角结论汇总到 Team Lead
  → 冲突检测（星级差 ≥2 或排名差 ≥3）
  → 综合研判 → 生成报告
  → 报告审计门（15% 抽样验证）
```

### 冲突检测规则

| 冲突类型 | 条件 | 标注 |
|---------|------|------|
| 星级分歧 | 最高星 - 最低星 ≥2 | "视角分歧大" |
| 排名分歧 | 进攻排名 - 防守排名 ≥3 | "进攻强/防守弱" 或反之 |

## 四、工具层

### 5 个工具一览

| 工具 | 用途 | 子命令 |
|------|------|--------|
| `financial_rigor.py` | Decimal 精度验算 | `verify-scale`, `verify-valuation`, `cross-validate`, `benford`, `calc`, `three-scenario` |
| `report_audit.py` | 报告质量门 | `extract`, `verdict` |
| `data_validator.py` | 双源交叉验证 | `validate`, `batch` |
| `stock_screener.py` | 动量+质量筛选 | `screen`, `grade` |
| `ashare_data.py` | A 股实时数据 | `quote`, `financials`, `valuation`, `search` |

### 数据验证标准

| 偏差范围 | 可信度 |
|---------|--------|
| ≤1% | ✅ 可信 |
| 1-5% | ⚠️ 供参考 |
| >5% | ❌ 不可信，需人工核实 |

## 八、约束（铁律）

1. **穿透防重叠**：推荐组合前三基金行业重合度 ≤15%
2. **预算硬平衡**：定投金额 ≤ 月净储蓄额
3. **费率穿透**：每次推荐披露完整费率
4. **常识校验**：PE/PB 异常值标注
5. **财务预检**：无应急金/高息负债先劝阻
6. **再平衡机制**：季度回顾 + 偏离 >10% 触发
7. **数据双源**：关键数据点强制交叉验证
8. **报告审计**：15% 随机抽样验证
