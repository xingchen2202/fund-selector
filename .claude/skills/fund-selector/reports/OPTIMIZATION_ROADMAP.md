# Fund Selector v2.0 — 优化建议清单（三轮尽调汇总）

> **日期**：2026-07-21 · **角色**：A 股基金经理 / 个人投资者 / 量化基金 PM
> **基础**：三轮深度尽调（121 测试全绿）+ 7 项 P0/P1 缺陷修复后的现状评估

---

## 〇、现状定位（重要）

经过三轮测试与修复，Fund Selector 当前的定位应明确：

> **它是一个"投研研究辅助工具"，不是"自动决策系统"。**

这个定位决定了优化方向：**提升辅助决策的质量、可信度、完整性**，而非追求全自动。以下建议按**投资价值提升优先级**排序。

---

## 一、P0 — 直接影响投资决策质量（建议 1 周内完成）

### 1.1 约束校验器集成到 skill 定义

**现状**：`constraint_validator.py` 是独立工具，19 个 skill 无一个强制调用。

**风险**：LLM 可能忘记调用，导致"8 条铁律"在某些场景下形同虚设。

**建议**：在 SKILL.md 增加强制条款：
```
## 强制约束校验
每次生成推荐组合后，必须调用 constraint_validator.validate_constraints()。
不通过则必须修正后重新校验，直至通过方可输出。
```

**投资价值**：⭐⭐⭐⭐⭐ — 这是核心承诺的"最后一公里"。

### 1.2 综合器否决后的备选方案

**现状**：全部基金被否决时 `top_pick=None`，用户看到"都被否决"但不知道"退而求其次选什么"。

**建议**：综合器增加 `fallback_candidate` 字段：
```json
{
  "top_pick": null,
  "fallback_candidate": {
    "code": "B",
    "name": "B",
    "avg_stars": 3.5,
    "note": "虽未达最优，但无1星风险，可考虑"
  }
}
```

**投资价值**：⭐⭐⭐⭐ — 提升用户体验，避免"分析半天无结果"。

### 1.3 约束不通过时的自动修复建议

**现状**：constraint_validator 只说"制造业重合 20% > 15%"，未说"怎么办"。

**建议**：增加 `suggestions` 返回字段：
```python
return {
    "passed": False,
    "failures": [...],
    "warnings": [...],
    "suggestions": [  # 新增
        "制造业重合 20%：建议将基金B 替换为科技/消费行业基金",
        "定投超额：建议将月投金额从 ¥5,000 降至 ¥3,000（月净储蓄额）"
    ]
}
```

**投资价值**：⭐⭐⭐⭐ — 从"诊断工具"升级为"处方工具"。

## 二、P1 — 提升投研完整性与专业度（建议 2 周内完成）

### 2.1 工作流增加"压力测试"环节

**现状**：FUND_ANALYSIS_WORKFLOW 4 步流程缺少极端行情模拟。

**建议**：在 Step 4（执行计划）后增加 Step 5：
```
## Step 5: 压力测试与最坏情景

基于历史极端行情（2015 股灾、2020 疫情、2022 下跌）：
- 组合最大可能回撤估计
- 最坏情景下的定投承受能力
- 极端行情触发条件（如"若组合浮亏 >25%，启动应急预案"）
```

**投资价值**：⭐⭐⭐⭐ — 这是专业投研与散户分析的核心差异。

### 2.2 推荐组合的相关性警告

**现状**：穿透分析检查行业重合，但未计算**持仓股票层面的相关性**。

**建议**：增加相关性检查：
```
若两只基金的前十大重仓股有 ≥3 只相同 → 警告"持仓高度重叠，分散效果有限"
```

**投资价值**：⭐⭐⭐ — 避免"看似分散、实则集中"的伪分散。

### 2.3 增加"策略回测"可选环节

**现状**：推荐的 DCA/金字塔/网格策略无历史表现验证。

**建议**：增加可选的 Step 6：
```
## Step 6: 策略回测（可选）

基于近 3-5 年历史净值，回测推荐策略：
- DCA 定投的年化收益 / 最大回撤 / 夏普比率
- 与一次性买入的对比
- 最坏入场时机的表现
```

**投资价值**：⭐⭐⭐ — 用历史数据增强说服力。

### 2.4 5 个 skill 标准化段落标题

**现状**：fund-checklist/fund-team/industry-funnel/quality-screen/thesis-tracker 用领域标题。

**建议**：保留领域标题，增加标准二级标题：
```markdown
## 流程
### 6 关检查流程
（原有内容）
```

**投资价值**：⭐⭐ — 提升自动化审计能力，便于后续工具链解析。

## 三、P2 — 技术债务与可维护性（建议 1 个月内完成）

### 3.1 MCP 集成测试

**现状**：AKShare 返回契约未测试（本会话无法连接 MCP）。

**建议**：
- 编写 `test_mcp_contract.py`，mock AKShare 返回，测试工具对异常返回的处理
- 覆盖：空 DataFrame、字段缺失、API 超时、签名变更

**技术价值**：⭐⭐⭐⭐ — AKShare 接口变更频繁，契约测试是稳定性保障。

### 3.2 LLM 输出行为测试

**现状**：测试覆盖工具层，但未测试 skill 实际执行时 LLM 的输出行为。

**建议**：使用 skill-creator 框架，设计 3-5 个真实 prompt：
- "推荐 3 只稳健型基金，月投 3000"
- "分析中欧新趋势混合"
- "我的持仓需要再平衡吗"

断言：输出包含必要段落、约束校验被调用、免责声明存在。

**技术价值**：⭐⭐⭐⭐ — 这是"工具正确"到"系统正确"的关键一跃。

### 3.3 CI 集成（GitHub Actions）

**现状**：测试需手动运行。

**建议**：增加 `.github/workflows/test.yml`：
```yaml
on: [push, pull_request]
jobs:
  test:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4
      - run: python .claude/skills/fund-selector/tests/tools/test_tools.py
      - run: python .claude/skills/fund-selector/tools/test_constraint_validator.py
      # ... 全部测试
```

**技术价值**：⭐⭐⭐ — 防止回归。

### 3.4 从"事后校验"到"生成中约束"

**现状**：constraint_validator 是推荐生成后的校验。

**建议（长期）**：将约束逻辑嵌入推荐生成过程：
- 生成候选池时即过滤违规项
- 资金分配时即检查预算平衡
- 输出前最终校验（双保险）

**技术价值**：⭐⭐⭐ — 效率提升 + 安全性增强。

## 四、P3 — 机构级能力（长期愿景）

### 4.1 多数据源适配（Wind/Choice）

**现状**：仅依赖 AKShare（免费源，质量不稳定）。

**建议**：抽象数据层，支持多源切换：
```
data_source:
  primary: akshare
  fallback: wind  # 或 choice, tushare
```

**机构价值**：⭐⭐⭐⭐ — 机构用户准入门槛。

### 4.2 真多 Agent 并行

**现状**：4 大师是"顺序读提示"（文档驱动），非独立推理。

**建议**：使用 Agent 工具启动 4 个独立子 Agent：
```
await agent("value_agent", ...)
await agent("growth_agent", ...)
# 各自独立上下文 → 真正独立判断 → 汇总
```

**技术价值**：⭐⭐⭐ — 真正的对抗式分析。

### 4.3 组合风险模型（VaR/CVaR）

**现状**：VaR 计算在旧架构 `calc_var_impact.py` 中，新架构未集成。

**建议**：集成到 constraint_validator 或独立模块：
- 基于持仓股票的历史波动率
- 计算组合 95%  VaR
- 超过风险预算 → 否决

**机构价值**：⭐⭐⭐⭐ — 量化风控核心能力。

### 4.4 持仓跟踪与漂移检测自动化

**现状**：thesis-drift 和 thesis-tracker 是静态 skill。

**建议**：增加自动化触发：
- 每周自动获取最新净值
- 自动检测风格漂移（持仓 vs 宣称风格）
- 基金经理变更自动预警

**投资价值**：⭐⭐⭐ — 从"单次分析"到"持续跟踪"。

## 六、一句话总结

**当前状态**：工具层优秀（A），约束层可用（B+），但约束的"最后一公里"（集成到 skill + 修复建议）尚未完成。

**最高 ROI 的 3 件事**：
1. 约束校验器集成到 SKILL.md（0.5 天，核心承诺落地）
2. 约束修复建议（1 天，从诊断→处方）
3. 否决后备选方案（0.5 天，用户体验质变）

**做完这 3 件，Fund Selector 从"可用的辅助工具"升级为"可信赖的辅助工具"。**
